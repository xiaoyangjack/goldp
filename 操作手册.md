# GoldQuant 操作手册

## 项目概述

GoldQuant 是一个专业级量化投研与交易平台，为A股个人/机构投资者提供一站式量化投研与模拟交易服务。

## 系统架构

- **前端技术栈**：Vue3 + TypeScript + Element Plus + ECharts + Vue Router + Pinia
- **后端技术栈**：Express.js + Node.js
- **测试框架**：Vitest + Vue Test Utils

## 快速开始

### 环境要求

- Node.js 16+
- npm 或 pnpm

### 安装步骤

1. **克隆或下载项目到本地**

2. **安装前端依赖**
```bash
cd frontend
npm install
```

3. **安装后端依赖**
```bash
cd backend
npm install
```

### 启动服务

#### 方式一：分别启动（推荐开发使用）

1. **启动后端服务**
```bash
cd backend
npm start
```
后端服务将运行在 http://localhost:3001

2. **启动前端服务（新开一个终端）**
```bash
cd frontend
npm run dev
```
前端服务将运行在 http://localhost:3000

#### 方式二：使用 pm2 同时启动（生产推荐）

1. 全局安装 pm2
```bash
npm install -g pm2
```

2. 启动后端
```bash
cd backend
pm2 start server.js --name goldquant-backend
```

3. 启动前端
```bash
cd frontend
npm run build
pm2 serve dist 3000 --name goldquant-frontend
```

## 开发指南

### 添加新页面

1. 在 `frontend/src/views/` 目录下创建新的 Vue 组件
2. 在 `frontend/src/router/index.ts` 中添加路由配置
3. 在 `frontend/src/components/Sidebar.vue` 中添加菜单项

### 添加新API

1. 在 `backend/server.js` 中添加新的路由处理
2. 在前端中调用 API 接口

### 样式规范

- 主色调：专业金融蓝 `#165DFF`
- 辅助色：低饱和度合规配色
- 使用 Element Plus 组件库
- 响应式设计，适配移动端到 4K 大屏

## 常见问题

### Q: 前端服务启动失败怎么办？

A: 请检查端口 3000 是否被占用，可以修改 `frontend/vite.config.ts` 中的端口配置。

### Q: 后端服务启动失败怎么办？

A: 请检查端口 3001 是否被占用，可以修改 `backend/server.js` 中的端口配置。

### Q: 页面显示 404 怎么办？

A: 请检查路由配置是否正确，确保页面文件存在且路由配置正确。

### Q: 如何修改模拟数据？

A: 可以修改 `backend/server.js` 中的 `mockData` 对象来修改模拟数据。
